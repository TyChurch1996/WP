Client Diagnosis

After analystist of the provided website i have identified the following issues:

GENERAL ISSUES
  • Poorly Handled Navigation(navigation uses multiple pages opposed to javascript).
  • Mass Amounts of Spaghetti code(unused or unneeded code).
  • Additionally css styling contains mass spaghetti code and mass duplicated code/reused code causing further performance issues.
  • Pointless xml and javascript code which performs additional functions however is irrelevant to the website and has shows no result to user or website.
  • Xml namespaces are implemented but not needed or used.
  • Paragraph headings poorly setup(paragraph headings cannot be identified easilly).
  • Poor Spacing between paragraphs(makes it harder to distinguish between headers and paragraphs).
  • The html/css code used on this website is poorly formatted from a development perspective and clusters multiple tags together on single lines
    (this makes further improvement on the website or updating comments more time consuming as finding elements on the website takes longer to find).
  • Duplicate Tags are used (multiple of the same headers).
  • Unneeded font tags(resizing font to the same size when its already set to that size).
  • Header placed off centre resulting in an awkward looking Header.
  • Poorly Chosen colour Scheme.
  • Inline spans placed outside of paragraphs.
  • Large bodies of text outside of paragraphs.
  • Newlines have been given closing tags (not required and creates spaghetti code).
  • Contains un-needed timers which is possible also depreciated code which was left in.
  • No disability option for people who are colour blind which violates the australian web accessibility laws and policies.

The issues above unless updated and corrected will cause much slower navigation of the website when loading webpages, increased stress on the server as more data is being processed, increased time required to change information on the website due to the spaghetti code currently in-place, Increase the eye Strain of users when viewing the website due to the colour theme, Create a higher power consumption by the server as its processing more data as well as most importantly violate the australian accessibility laws and policies as colorblind people will have difficulty navigating the website as it has not been built to accomodate for people requiring aid with there vision.

Client Proposal

My proposal to the client would be to recreate the website from scratch since the code it litered with spaghetti code it would be more time efficient to recreate the website from scratch and move the previously stored data to the recreated website.

In order to recreate the website we have to address the previously mentioned issues, the proposed methods for dealing with these would be to:

Firstly Restructure the navigation pane to use javascript as this will allow for instantaneous webpage navigation since the page format remains the same throughout the entire web site, by doing this it should make website navigation instantanious as long as the rest of the website maintains good programming practices.

The next stage for updating the web site would be to recreate the css styling for the webpage as well as rewrite the html in html5 and implement a better theme which increases readibility and does not place additional strain on the users eyes when implementing this theme we also have to either implement an additional style or make the current style viable for people with disabilities in order for the site to be legally viable.

Colour blindness and most other vision disabilities css styling can be switched between easilly with the use of a button if the current style is not implemented in a manor which supports these disabilities.

When implementing our header we can centre it with css styling which will give the website a much nicer feel and increase readibilty due to the heuristic design principle recognition rather than Recall which essentially means you want your users to recognise how to navigate rather then learn where things are then have to remember that later.

When recreating the intro page we can implement an alignment style tag within the above paragraphs of the page, doing so will ensure our paragraph content starts and stops at the correct place, we can also limit how long we want our sentances by adjusting the width of these paragraphs in a style which neatly lines up all content.

Below this we have a list of abbreviations and terms, here we can use a html5 unorder list to ensure the content entered is aligned nicely and increases user readability while letting the user extract the information needed with ease.

When Recreating the Places page we can Centre and underline the header to make it stand out from the content while we can use a ordered or unordered list to list all places mentioned in a neater fashion, by doing this the web page will become more user friendly and readable as well as align the content, without this being implemented the current state is quite hard to read and poorly optimized.

The final page with large changes required is the comments section which currently requires someone to manually update this page , with the usage of javascript and php we can update the comments section to be automated and user friendly, it is highly recommended to update this section as this is the websites primary feedback source for everything from content recommendations to user errors, currently an email is provided on this page but only as a text styled implementation, an email can still be provided on this page for other enquiries however it is highly recommended to update the text style to an email style to allow users to click an email link and preload the option to send an email to the provided email address.

Since we are recreating the website from scratch this removes the need to remove the previously added spaghetti code and will dramatically decrease our development time as we will no longer have any performance issues and all code updated will be specific to the task at hand with the option to expand further without additional hurdles.
